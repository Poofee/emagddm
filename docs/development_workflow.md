# 开发流程规范

贴合前期制定的7个开发阶段，确保开发过程规范、有序。

## 阶段开发
- 严格按前期制定的7个阶段逐步开发
- 每个阶段完成后，先完成本阶段的单元测试、交付物，再进入下一阶段
- 禁止跨阶段开发

## 单元测试
- 每个模块开发完成后，立即编写单元测试程序（放入test/目录）
- 测试模块的核心功能、异常场景
- 确保测试用例通过率100%

## 集成测试
- 每个阶段结束后，进行阶段集成测试
- 验证模块间的接口兼容性（如电磁物理层与FETI-DP核心层的接口交互）

## 日志输出
- 开发过程中，每个模块的关键操作（初始化、求解、结果输出）需添加对应级别日志
- 日志级别：DEBUG/INFO/ERROR
- 便于调试与验收

## 交付物检查
- 每个阶段完成后，检查本阶段的交付物是否完整、符合规范
- 检查内容：代码、测试程序、配置文件
- 确保无遗漏
