# 工业级低频电磁有限元求解器核心架构开发AI提示词
## 一、基础身份与任务背景声明
你是一位拥有15年以上工业级电磁仿真软件（ANSYS Maxwell、JMAG、COMSOL Multiphysics）核心开发经验的资深软件架构师，精通低频电磁场有限元方法的数学理论与工程实现，熟悉C++高性能数值计算、模块化软件架构设计、并行计算与工业级软件鲁棒性要求。

我已完成低频电磁有限元求解器的**底层基础模块**开发，包括：
- 线性代数核心：基于Eigen的稠密/稀疏矩阵运算，集成MUMPS、SuperLU等直接求解器接口
- 网格数据结构：支持三角形、四边形（二维）、四面体、六面体（三维）单元的拓扑表示与遍历
- 基础形函数库：1阶/2阶拉格朗日单元、1阶/2阶棱边单元（Nédélec）的形函数与雅可比矩阵计算
- 材料数据管理：基本的线性/非线性铁磁材料、导体、绝缘体、永磁体的参数存储与插值
- 基础数值工具：数值积分（高斯积分）、边界积分、矩阵装配基础框架

**核心任务**：基于上述已完成的底层模块，设计并输出一套**工业级、模块化、可扩展、高性能**的低频电磁有限元求解器**核心架构**，全面支持二维、三维、轴对称模型的电场、磁场求解，完整覆盖边界条件、激励、绕组、运动等工业核心功能，并提供可直接落地的模块接口定义、数据流转方案与核心代码骨架。

## 二、核心架构设计总要求
1. **模块化与解耦**：采用分层架构+插件式设计，各模块之间通过抽象接口通信，实现"高内聚、低耦合"，支持后续扩展新的物理场、单元类型、求解器与边界条件
2. **维度统一**：设计统一的核心框架，通过"维度特化"机制实现二维、三维、轴对称模型的代码复用，避免重复开发
3. **工业级鲁棒性**：全面考虑数值稳定性、错误处理、边界条件冲突检测、非法输入处理、内存泄漏防护
4. **高性能优先**：支持多线程并行计算（OpenMP）、分布式内存并行（MPI预留接口）、稀疏矩阵优化、内存池管理
5. **可调试性与可测试性**：内置详细的日志系统、性能统计模块、单元测试框架接口
6. **前后处理友好**：设计标准化的数据接口，便于后续集成网格导入/导出、结果可视化、参数化分析功能

## 三、必须详细设计的核心模块与功能点
### （一）物理场抽象与求解器调度模块
1. 设计统一的`PhysicsField`抽象基类，定义所有物理场的通用接口（装配矩阵、求解、后处理）
2. 派生出具体的物理场求解器类，至少包括：
   - 静电场求解器（二维/三维/轴对称）
3. 设计`SolverScheduler`类，负责求解流程的统一调度：
   - 前处理检查（网格有效性、边界条件完整性、材料参数合理性）
   - 自由度（DOF）分配与管理（核心！需支持节点自由度、棱边自由度、面自由度的混合分配）
   - 矩阵与右端向量的初始化与清零
   - 边界条件与激励的施加
   - 非线性迭代控制（牛顿-拉夫逊法）
   - 时间步进控制（瞬态求解）
   - 求解结果的存储与后处理触发
4. 明确各物理场的控制方程、未知量类型与自由度分配规则，特别是三维棱边单元（Nédélec）的自由度管理与矩阵装配

### （二）自由度（DOF）管理核心模块
1. 设计通用的`DOFManager`类，支持：
   - 不同维度、不同单元类型、不同物理场的自由度统一编号
   - 约束自由度（Dirichlet边界条件）的标记与处理
   - 多物理场耦合的自由度映射
   - 并行计算中的自由度分区与通信
2. 实现"自由度压缩"机制，自动排除约束自由度，减小求解矩阵规模
3. 提供自由度与几何实体（节点、棱边、面、体）的双向映射接口

### （三）边界条件与激励管理模块
1. 设计统一的`BoundaryCondition`抽象基类，定义所有边界条件的通用接口（施加到矩阵、施加到右端向量）
2. 派生出低频电磁常用的边界条件类，至少包括：
   - Dirichlet边界条件（电位、磁标位、磁矢位）
   - Neumann边界条件（电场法向分量、磁场切向分量）
   - 混合边界条件（阻抗边界、辐射边界）
   - 周期性边界条件（平移周期、旋转周期）
   - 绝缘边界、理想电壁、理想磁壁
3. 设计统一的`Excitation`抽象基类，派生出：
   - 电压源激励（直流/交流/瞬态）
   - 电流源激励（直流/交流/瞬态）
   - 永磁体激励（剩磁、矫顽力）
   - 外电路激励（预留接口，支持后续集成电路求解器）
4. 实现边界条件与激励的优先级管理、冲突检测与自动合并机制
5. 支持随时间、随空间变化的边界条件与激励

### （七）数据处理与结果输出模块
1. 设计统一的`FieldData`类，存储所有求解结果数据（节点值、单元值、积分点值）
2. 实现常用物理量的后处理计算：
   - 电场强度、电位移矢量、电流密度
   - 磁场强度、磁感应强度、磁通量
3. 支持结果数据的插值、积分、微分运算
4. 设计标准化的结果输出接口，支持导出为VTK、CSV等通用格式
5. 实现内存高效的结果数据管理，支持大型模型的增量式输出

## 四、关键技术实现要求
1. **轴对称模型特殊处理**：详细说明轴对称模型（r-z坐标系）的控制方程推导、形函数修改、积分权重调整、边界条件转换方法，确保与二维/三维框架的无缝集成

3. **矩阵装配优化**：设计高效的单元矩阵计算与全局矩阵装配流程，支持多线程并行装配，避免重复计算
4. **内存优化**：使用内存池管理动态分配的内存，采用稀疏矩阵的压缩存储格式（CSR），支持大型模型的内存需求
5. **错误处理**：实现分级错误处理机制（警告、错误、致命错误），提供详细的错误信息与定位，支持异常安全的代码编写


5. **核心代码骨架**：提供至少3个核心类（`DOFManager`、`SolverScheduler`、`Winding`）的完整C++头文件与关键实现代码
6. **性能优化建议**：针对工业级大型模型（100万自由度以上）的性能瓶颈，提出具体的优化方案
7. **测试方案**：设计核心模块的单元测试用例与整体求解器的验证用例（与ANSYS Maxwell结果对比）
